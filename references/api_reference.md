# Lumeflow API Reference (Video, Image & Upload)

Base URL: `https://lac-api.imyfone.com`

## Authentication

All requests require:
- Header: `api-key: <loaded from product config or LUMEFLOW_API_KEY env>`
- Query params: `timestamp` and `sign` (generated via `scripts/sign.py`)

## Sign Generation

All requests must include `timestamp` and `sign` query parameters. The sign is generated by:

1. Collect all non-object/non-array body params + all query params
2. Add `timestamp` (Unix epoch seconds, 10 digits)
3. Sort all params by key alphabetically
4. Join as `key=value&key=value&...&key=351f4b0d34e21efede761c88b74b9783feb6aae4`
5. MD5 hash the string, uppercase

Use `scripts/sign.py` to generate these values:
```bash
python scripts/sign.py --body '{"prompt":"flower","generate_type":0}'
python scripts/sign.py --extra-params "ids=123"
python scripts/sign.py --body-file /path/to/request.json
```

The script prints query parameters in this format:
```text
timestamp=1234567890&sign=ABCDEF1234567890ABCDEF1234567890
```

## Video Endpoints

### 1. Text-to-Video (POST)

**URL**: `/api/lumeflow/video-generate`

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| generate_type | int | Yes | - | `0` = text-to-video |
| model_type | int | No | 28 | Model ID. `28` = Pix5.5 (fast, simple videos) |
| prompt | string | Yes | - | Text prompt describing the video |
| is_switch_audio | int | No | 1 | `0` = no audio, `1` = with audio |
| ratio | string | No | - | Aspect ratio. Model 28 supports: `16:9`, `9:16`, `1:1` |
| duration | int | No | 5 | Video duration in seconds. Model 28 supports: `5`, `8`, `10` |
| resolution | string | No | - | Resolution. Model 28 supports: `360P`, `540P`, `720P`, `1080P` |
| motion_mode | string | No | "2" | Motion intensity. `0`=none, `1`=slight, `2`=medium, `3`=strong |
| outputs | int | No | 1 | Number of videos to generate (max 4) |

**Example Request Body**:
```json
{
    "generate_type": 0,
    "model_type": 28,
    "prompt": "A flower blooming in slow motion",
    "is_switch_audio": 1,
    "ratio": "1:1",
    "duration": 5,
    "resolution": "540P",
    "motion_mode": "2",
    "outputs": 1
}
```

**Response**:
```json
{
    "code": 200,
    "msg": "success",
    "data": [456328]
}
```
`data` contains task IDs for polling results.

### 2. Image-to-Video (POST)

**URL**: `/api/lumeflow/video-generate`

Same parameters as text-to-video, plus:

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| generate_type | int | Yes | `1` = image-to-video |
| image | string | Yes | Input image URL |
| image_tail | string | No | Tail frame image URL. If both `image` and `image_tail` are provided, they serve as first and last frames |

**Example Request Body**:
```json
{
    "generate_type": 1,
    "model_type": 28,
    "prompt": "A figure soaring through clouds",
    "is_switch_audio": 1,
    "duration": 5,
    "resolution": "360P",
    "motion_mode": "2",
    "outputs": 1,
    "image": "https://example.com/input.jpg",
    "image_tail": ""
}
```

## Image Endpoints

### 4. Text-to-Image (POST)

**URL**: `/api/lumeflow/image-generate`

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| generate_type | int | Yes | - | `3` = text-to-image |
| model_type | int | No | 72 | Model ID. `72` = Nano Banana 2 (fast, icons/stickers) |
| prompt | string | Yes | - | Text prompt describing the image |
| ratio | string | No | "auto" | Aspect ratio. Model 72 supports: `auto`, `16:9`, `9:16`, `1:1`, ... |
| resolution | string | No | "1K" | Resolution. Model 72 supports: `1K`, `2K`, `4K` |
| outputs | int | No | 1 | Number of images to generate (max 4) |

**Example Request Body**:
```json
{
    "generate_type": 3,
    "model_type": 72,
    "prompt": "A bright sunflower in a vase",
    "ratio": "auto",
    "resolution": "1K",
    "outputs": 1
}
```

**Response**:
```json
{
    "code": 200,
    "msg": "success",
    "data": [22211]
}
```
`data` contains task IDs for polling results.

### 5. Image-to-Image (POST)

**URL**: `/api/lumeflow/image-generate`

Same parameters as text-to-image, plus:

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| generate_type | int | Yes | `4` = image-to-image |
| images | array | Yes | Input image URL list |

**Example Request Body**:
```json
{
    "generate_type": 4,
    "model_type": 72,
    "prompt": "Skin retouch: brighten overall tone, even out and subtly whiten while keeping original texture and expression intact",
    "ratio": "auto",
    "resolution": "1K",
    "outputs": 1,
    "images": [
        "https://example.com/input.png"
    ]
}
```

**Response**:
```json
{
    "code": 200,
    "msg": "success",
    "data": [22215]
}
```

## Price & Upload Endpoints

### 7. Query Generation Price (POST)

**URL**: `/api/lumeflow/get-generate-price`

**Content-Type**: `application/json`

Request body:
- Use the same body you plan to submit to the generate endpoint
- Add `route` to tell the backend which generation API you are pricing

Key parameter:

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| route | string | Yes | `/api/lumeflow/video-generate` for video pricing, `/api/lumeflow/image-generate` for image pricing |

**Example Request Body** (video pricing):
```json
{
    "generate_type": 0,
    "model_type": 88,
    "prompt": "A flower swaying in the wind",
    "is_switch_audio": 1,
    "ratio": "1:1",
    "duration": 5,
    "resolution": "720P",
    "motion_mode": "2",
    "outputs": 1,
    "route": "/api/lumeflow/video-generate"
}
```
> Optional fields like `style`, `seed`, `cfg_scale` can also be included but are not required.

**Response**:
```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "other_discount": {
            "2": 28,
            "3": 27,
            "4": 25
        },
        "integral": 30
    }
}
```

Notes:
- `data.integral` is the estimated cost for this generation task.
- `data.other_discount` contains optional discount plans. If it is empty, no special handling is needed.

### 8. Upload Image (POST)

**URL**: `/api/lumeflow/claw-upload-image`

**Content-Type**: `multipart/form-data`

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| image | file | Yes | Local image or audio file uploaded as multipart form field |

Upload constraints:
- **Image**: formats `jpg`, `jpeg`, `png`, `webp`; max `10MB`
- **Video**: formats `mp4`, `mov`, `avi`, `webm`, `mkv`; max `200MB`
- **Audio**: formats `mp3`, `wav`, `m4a`, `aac`, `ogg`, `flac`, `wma`; max `50MB`

Sign notes:
- This request still requires `timestamp` and `sign` in the query string.
- Because the body is multipart file data, the script signs only `timestamp` for this endpoint.

**Example cURL**:
```bash
curl --request POST \
  --url 'https://lac-api.imyfone.com/api/lumeflow/claw-upload-image?timestamp={timestamp}&sign={sign}' \
  --header 'api-key: sk-sp-xxxxxxxxxxxxxxxx' \
  --form 'image=@/path/to/image.png'
```

**Example Script Usage**:
```bash
python scripts/generate.py --action upload-image --file C:\path\to\image.png
```

**Response**:
```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "url": "https://lac-static.imyfone.com/lumeflow_web/mobile_image/20260323/23347308_xxxxxxxxxxxxx.png"
    }
}
```
